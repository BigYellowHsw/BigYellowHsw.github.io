python配置virtualenv环境

发表于 2018-12-23 | 分类于 python | 本文总阅读量 次

在python实际项目开发中，我们通常会根据自己的需求去下载各种相应的框架库，如Scrapy、Beautiful Soup等，但是可能每个项目使用的框架库并不一样，或使用框架的版本不一样，这样需要我们根据需求不断的更新或卸载相应的库。直接对我们的Python环境操作会让我们的开发环境和项目造成很多不必要的麻烦，管理也相当混乱。所以我们在项目开发中，应该对不同的项目建立不同的虚拟环境，下载安装该项目需要的python版本、各种依赖库，然后再进行开发。本文就简单讲解下如何配置python的虚拟环境。

python配置virtualenv环境有两种方式：一种是cmd命令创建虚拟环境再激活，另一种方法是利用pycharm设置里添加个虚拟环境。

第一种：cmd命令创建虚拟环境再激活

例如test文件夹中建立虚拟环境
pip install virtualenv

E:\>cd test

E:\test>virtualenv venv
Using base prefix 'c:\\python\\python35'
New python executable in E:\test\venv\Scripts\python.exe
Installing setuptools, pip, wheel...
done.

开启关闭virtualenv
进入venv目录
激活
cmd>E:\virtualenv\venv>Scripts\activate
关闭
(venv) E:\virtualenv\venv>deactivate

第二种：利用pycharm设置里添加个虚拟环境

在setting设置中查看环境设置

在pycharm下方找到terminal终端即可查看虚拟环境相关信息。默认是使用的虚拟环境，可以在setting设置中更改自己所需环境。
